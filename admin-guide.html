📖 Admin Guide

Step-by-step playbook for running an MBB Boxes pool

🏀

March Madness Boxes — How It Works

Players claim squares on a 10×10 grid. Numbers are randomly drawn before the tournament starts. After each game, the winning team's last score digit (column) and the losing team's last score digit (row) determine the winning box. Each round has a fixed payout — Play-In through Championship.

Phase 1 Create the Pool Before entries open
  • 1 Go to 🎲 Boxes Pools (boxes-pool.html) and click ➕ Create New Pool.
  • 2 Set Sport to College Basketball, Event to March Madness.
  • 3 Set Box Winner Logic to Winner's digit = Columns, Loser's digit = Rows (MBB style).
  • 4 Set Payout Mode to Fixed amount per tournament round and confirm the per-round amounts (Play-In $25 → Championship $500).
  • 5 Set Grid Numbers to Same numbers for all games.
  • 6 Set Payout Timing to Final only.
  • 7 Enter Cost Per Box and pool name, then click Create Pool.
💡 Tip: You don't need to add individual games yet — the pool grid works without them. Games are only needed to link scores later.
Phase 2 Collect Entries Before the tournament starts
  • 1 Share the link to 🎲 boxes-pool.html with participants. They sign in and click empty boxes to claim them.
  • 2 For anyone who can't self-enter, use the Admin Panel → Manual Entry tab on boxes-pool.html. Click any empty box directly on the grid to open the assign modal — no need to type coordinates.
  • 3 Track payments using the Paid / Unpaid toggle. Click a taken box on the grid as admin to toggle payment status at any time.
💡 Tip: The pool stats at the top of the page show claimed vs available boxes in real time. Share the page link — it updates live for everyone.
Phase 3 Draw Numbers & Lock the Pool Just before the first game tips off
  • 1 Open the Admin Panel → Games tab on 🎲 boxes-pool.html. A placeholder Game 1 is created automatically with the pool — click 🎲 Randomize Numbers to assign random digits, or type them manually into the Column Numbers and Row Numbers fields if the draw was done offline (comma-separated, e.g. 3,7,1,9,0,6,4,2,8,5). Since the pool uses shared numbers, this one set applies to all games.
  • 2 Check the Numbers Drawn checkbox so numbers become visible to players, then click 💾 Save All Games.
  • 3 Click 🔒 Lock Pool in Pool Settings. This freezes entries — no new claims or changes. The Results and Standings tabs become visible to all signed-in users.
⚠️ Locking is permanent — only unlock if there was a genuine error. Unlocking re-opens claims, which is confusing for participants. Do this right before tip-off.
Phase 4 During the Tournament — Settle Each Game After each game goes final
  • 1 Open 📡 Score Fetcher (admin-score-fetch.html). Select NCAAB Men, pick today's date, and click Fetch Scores.
  • 2 Finished games appear in the Settle Section with a ✓ status. Check the box next to the game and click Push Scores. This writes the final score to games/MBB/games in Firestore.
  • 3 Switch to 🎲 Admin Boxes (admin-boxes.html) → Update Scores. Select your pool.
  • 4 For each game card, open the ESPN Game dropdown, select the matching game, and click 🔗 Save Link (only needed once per game — it's remembered).
  • 5 Click ⬇ Import Final Score. The final scores auto-populate, the winning box is calculated, and results are saved to Firestore instantly.
  • 6 Results and Standings tabs on 🎲 boxes-pool.html update automatically for all users — no further action needed.
💡 Tip: The ESPN link only needs to be set once per game. After that, just hit Import Final Score each time. You can batch multiple games in a single session during busy round days.
⚠️ Team order matters: The importer tries to auto-match team names. If it can't, it shows a confirm dialog — read it carefully before clicking OK or Cancel, as it determines which team's score is the column digit vs the row digit.
Phase 5 Payouts & Wrap-Up After the Championship game
  • 1 Open the 🏅 Standings tab on boxes-pool.html — it shows each winner's total earnings across all games. The footer row shows grand total paid out.
  • 2 Cross-reference with payment status. Click any taken box on the grid to check if that person was marked Paid.
  • 3 Use the 📋 Results tab to verify every game's winner and payout before sending money — it shows the exact score, winning box, and dollar amount per game.
  • 4 Venmo winnings. The pool is complete — optionally archive by leaving it locked, or delete it via Admin Panel → Pool Settings → 🗑️ Delete Pool.

❓ FAQ

I ran the Score Fetcher but the Import button is still disabled. Why? ▼
The Import button enables only after you've selected a game from the ESPN Game dropdown and clicked Save Link. If you already set the link previously, re-select the pool in admin-boxes to reload — the link should auto-populate and the button should be active.
The score importer showed a team-order dialog. Which option should I pick? ▼
This happens when the importer can't auto-match the box pool team names to ESPN's away/home teams. Read the dialog carefully — it shows both options with their scores. In MBB-style pools, it doesn't actually matter which team is T1 vs T2 because the winner/loser logic automatically assigns the higher score to columns regardless. Just be consistent: pick the order that matches how you entered Team 1 and Team 2 when you set up the game.
A game isn't showing up in the Score Fetcher. What do I do? ▼
Try changing the date on the Score Fetcher to match the game date exactly — ESPN's API filters by date. If the game was played yesterday, select yesterday's date. If it still doesn't appear, the game may not be in ESPN's NCAAB Men scoreboard (e.g. it might be a women's game — check the sport selector). As a fallback, you can always enter the final score manually in admin-boxes without using the Import button.
Can I add a game to the pool after it's locked? ▼
Yes — adding games is handled separately from the lock. Unlock the pool briefly, go to Admin Panel → Games tab, add the game and save, then re-lock. The grid itself and all entries are unaffected. Alternatively, you can add games in admin-boxes directly without unlocking — just use the Games tab in boxes-pool.html admin panel which doesn't require the pool to be unlocked to edit game metadata.
Someone paid but is still showing as Unpaid. How do I fix it? ▼
As an admin, click their box directly on the grid on boxes-pool.html. A manage modal will appear showing their name and current payment status. Click Mark Paid to toggle it. The change saves immediately to Firestore.
A player wants to be removed / give up their box. How do I unclaim it? ▼
Unlock the pool first (if locked), then click their box on the grid as admin and choose 🗑 Unclaim. Confirm the prompt. The box returns to available. Re-lock the pool once done.
The Standings tab shows $0 for everyone even though I imported scores. Why? ▼
This usually means the round field on the game doc doesn't match the expected round key. The Score Fetcher stores rounds like round-of-64, sweet-16, etc. Check the game document in Firestore (games/MBB/games) and make sure the round value matches one of: play-in, round-of-64, round-of-32, sweet-16, elite-8, final-four, championship.
